1 - Pilares
    Tributos = Colunas da tabela, características do problema
    Instâncias = Linhas da tabela, acontecimento de cada ocorrencia
    Classes = Prever algo para análise, geralmente true ou false
    Tipos de dados = Categoricos ou Numericos

2 - Tarefas
    Classificação
        Matriz de confusão
        Formula de acuracia: (VP + VN) / (VP + VN + FP + FN)
        Formula de precisão: VP/(VP+FP)
        Formula de recall: VP/(VP+FN)
        Formula de especificidade: VN/(VN+FP)
        Formula de F1 Score: 2 * (Precisão * Recall) / (Precisão + Recall)
        Formula de AUC-ROC
    Regressão
        Formula Mean Error: Soma da diferença entre realizado e previsto / quantidade de elementos 
        Formula Mean Absolute Error: Soma da diferença entre realizado e previsto / quantidade de elementos (não há sinais de - ou +)
        Formula Root Mean Squared Error: Independe de grandezas, não sofre influencia pelo tamanho ou quantidade do valor
            raiz((diferença entre realizado e previsto elevado ao quadrado)/quantidade de elementos)
        Formula Mean Percentage Error: ((realizado - previso)/realizado * 100)/quantidade de elementos
        Formula Mean Absolute Error: Independe de grandezas, a formula é a mesma, sem sinais
    Agrupamentos
    Regras de associação
    Detecção de Anomalias
    Aprendizado por reforço
    Linguagem natural
    Redes neurais
    Redução de dimensionalidade
    Aorendizado semi-supervisionado

3 - Divisoes de dados
    Treino
    Teste
    Validacao

3 - Tecnicas de divisao de dados para classificação
    Holdout
        treino 70%
        teste 30%
    cross-validation
        test 10% x 10 vezes
    Leave-Out
        treino 90% - separado em 9 de 10%
        test 10%
    K-Fold
        k = quantidade de % decidida
        treinar no k-1
        testar no k restante
    Subasmotragem
        treino 70% com dados randomicos
        teste 30% com dados randomicos

4 - Categorical Encoding
    Processo de transformar dados categóricos em dados númericos para possibilitar os calculos para previsão
    Pré-Processamento:
        Label Encoding: 
            Cada categoria recebe um valor para ter a correlação
            Pode haver influência de grandezas
            Usado quando há order ou quando possuimos grande quantidade de categorias
        One-Hot Encoding:
            Cada categoria é transformada em outro atributo, dummy variable
            Cada elemento do atributo recebe um valor binário de acordo com a quantidade de elementos dentro do atributo
            Usado quando não há ordem ou quando o número de atributo é pequeno

5 - Feature Scale - Dimensionamento de características
    Processo de transformação de dados númericos, simplifica-los para o cálculo
    Variáveis em escala diferentes
    Formula Padronização Z-Score: Dados próximos a uma distribuição normal, existe outliers
    Formula Normalização Min-Max: Quando não sabemos a distribuição, desconsidera outliers

6 - Agrupamentos
    Usado quando não supervisionado
    Necessário criar grupos de classes sobre as atribuições
        Agrupamento completo
        Agrupamento parcial
        Agrupamento difuso
        Agrupamento hierarquico
        O que não é agrupado é caracterizado como ruído
    k-means e k-medoid
        baseado em protótipos
        grupos definidos
        define o centróides (pontos iniciais)
        utiliza distância euclidiana
        k-means
            protitipado em cima de centróides, média de grupo de pontos
        k-medoid
            prototipado em cima de medóides, ponto mais representativo
    DBSCAN
        baseado em densidade
        numero de grupos definidos automaticamente
        a densidade é baseada no raio especificado
    hierarquico
        baseado em aglomeração um grupo
        baseado em divisiva um outro grupo
    
7 - Regras de associação
    estabelece um minimo de suporte, confiança e lift de associação de regra
    apriori e fp-grow são algoritmos de redes neurais usada para regras de associaçao

8 - Regressão Linear
    eixo y = variavel de resposta ou dependente
    eixo x = variavel explanatoria ou independente

    calculo de correlação
        valor entre -1 e 1
        força e direção
    
    coeficiente de determinação
    r ao quadrado

    ponto de intersecção é o encontro da linha no eixo Y, onde o X=0
    inclinação é o ângulo, a cada aumento de X o Y aumenta

    calculos de previsão:
        p = intersecção + (inclinação * valor a prever)
    
    calculos de residuais:
        histograma
        diagrama de normalidade
        teste de shapiro-wilk
    
    simples: uma variavel explanatoria para uma variavel dependente
        ex: plano de saude, idade para identificar valor de custo
    
    multipla: duas ou mais variaveis explanatoria para uma variavel dependente
        ex: desempenho de um carro, tenho variavel de combustivel, pressão dos pneus, temperatura do motor, etc
        aqui é necessario calcular cada regressão entre x e y
        aqui usamos o r ao quadrado ajustado

        colinearidade são duas variaveis independentes que possuem correlação
        parcimonia é o ato de evitar o uso de váriaveis que não afetam o modelo em nada

        sequência de cálculo, pois possuem uma dependência entre eles:
            correlação > inclinação > intersecção > previsão

        correlação de person para termos o valor da correlação
        inclinação = correlação(desvio padrão de X / desvio padrão de Y)
        intersecção = média de Y - (inclinação * média de X)
        previsão = intersecção + (inclinação * variável independente)
